MercadoLibre Chile scrapers

Measure MercadoLibre Chile saturation: local vs imported listings

A normal MercadoLibre Chile search page can mix local sellers with cross-border listings. For product validation, only local Chile stock should count as local competition.

Recommended actor

Use Atomic Actor 1 for saturation: it coordinates child workers, avoids duplicate mission work, and reports cost/block telemetry.

Input

{
  "atomicMode": "orchestrator",
  "searchQueries": ["monitor de bebe", "propane gauge"],
  "searchFilterMode": "envio_local",
  "maxItems": 120,
  "maxItemsPerQuery": 60,
  "maxPagesPerQuery": 1,
  "useMissionQueue": true,
  "missionDedupeMode": "mission",
  "resumeMission": true,
  "maxConcurrentChildRuns": 4,
  "proxyConfiguration": {
    "useApifyProxy": true,
    "apifyProxyGroups": ["RESIDENTIAL"],
    "apifyProxyCountry": "CL"
  }
}

What to measure

  • Number of rows with `local_stock_status` local.
  • Number of rows with `local_stock_status` import_only.
  • Number of rows with unknown evidence.
  • `search_total_results` and `search_local_filter_total_results` when exposed by MercadoLibre.
  • Price distribution among local rows only.

Failure rule

If scraping is blocked or visibility is unknown, preserve unknown competition. Do not turn missing data into a no-competition signal.
